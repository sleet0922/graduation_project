# Go 项目详细讲解：从入门到精通

## 一、基础概念讲解

### 1. 什么是接口 (Interface)

**接口**是一组方法签名的集合，它定义了一个对象应该具有的行为。

**语法**：
```go
type 接口名 interface {
    方法名1(参数1 类型1, 参数2 类型2) 返回值类型
    方法名2(参数 类型) 返回值类型
    // 更多方法...
}
```

**作用**：
- 定义行为规范
- 实现多态
- 解耦组件

### 2. 什么是结构体 (Struct)

**结构体**是字段的集合，用于封装数据。

**语法**：
```go
type 结构体名 struct {
    字段名1 类型1
    字段名2 类型2
    // 更多字段...
}
```

**作用**：
- 封装数据
- 实现接口方法
- 组织代码结构

### 3. 什么是构造函数 (Constructor)

**构造函数**是创建并返回结构体实例的函数。

**语法**：
```go
func New结构体名(参数 类型) *结构体名 {
    return &结构体名{
        字段1: 值1,
        字段2: 值2,
    }
}
```

**作用**：
- 初始化结构体
- 注入依赖
- 提供创建实例的标准方式

### 4. 什么是依赖注入 (Dependency Injection)

**依赖注入**是将依赖的对象通过构造函数或方法注入，而不是在内部创建。

**作用**：
- 降低耦合
- 便于测试
- 提高代码可维护性

## 二、项目创建顺序详解

### 第一步：创建数据模型 (Model)

**文件**：`internal/model/user.go`

```go
package model

import "gorm.io/gorm"

// User 用户模型
type User struct {
    gorm.Model           // 嵌入 gorm.Model，包含 ID、CreatedAt、UpdatedAt、DeletedAt 字段
    Name       string `json:"name" gorm:"not null"`       // 用户名，json 标签表示序列化时的字段名，gorm 标签表示数据库约束
    Account    string `json:"account" gorm:"uniqueIndex;not null"`  // 账号，唯一索引
    Password   string `json:"-" gorm:"not null"`          // 密码，json:"-" 表示序列化时忽略该字段
    Phone      string `json:"phone" gorm:"uniqueIndex;not null"`    // 手机号，唯一索引
    Avatar     string `json:"avatar" gorm:"default:''"`    // 头像，默认空字符串
    Gender     int    `json:"gender" gorm:"default:0"`     // 性别，默认 0
    Birthday   string `json:"birthday" gorm:"default:''"`   // 生日，默认空字符串
    Location   string `json:"location" gorm:"default:''"`   // 位置，默认空字符串
    UserStatus int    `json:"user_status" gorm:"default:0"` // 用户状态，默认 0
}

// TableName 指定表名
func (User) TableName() string {
    return "user" // 表名为 user
}
```

**讲解**：
- `type User struct`：定义了一个名为 User 的结构体
- `gorm.Model`：嵌入了 GORM 的基础模型，自动包含 ID、CreatedAt、UpdatedAt、DeletedAt 字段
- 每个字段都有类型、json 标签和 gorm 标签
- `TableName()` 方法：指定数据库表名

### 第二步：创建数据访问层 (Repository)

**文件**：`internal/repo/user_repo.go`

```go
package repo

import (
    "sleet0922/graduation_project/internal/model"

    "gorm.io/gorm"
)

// UserRepository 数据访问接口
type UserRepository interface {
    Add(user *model.User) error                  // 添加用户
    Delete(id uint) error                        // 删除用户
    Update(user *model.User) error               // 更新用户
    DeleteAll() error                            // 删除所有用户
    AddTestUser() error                          // 添加测试用户
    GetByID(id uint) (*model.User, error)        // 根据 ID 获取用户
    GetByAccount(account string) (*model.User, error)  // 根据账号获取用户
    GetByPhone(phone string) (*model.User, error)      // 根据手机号获取用户
    UpdateAvatar(userID uint, avatar string) (*model.User, error)  // 更新头像
    UpdateName(userID uint, name string) (*model.User, error)      // 更新用户名
}

// userRepository 数据访问实现
type userRepository struct {
    db *gorm.DB  // 依赖 gorm.DB 实例
}

// NewUserRepository 创建 UserRepository 实例
func NewUserRepository(db *gorm.DB) UserRepository {
    return &userRepository{db: db}  // 注入 db 依赖
}

// Add 添加用户
func (r *userRepository) Add(user *model.User) error {
    return r.db.Create(user).Error  // 调用 gorm 的 Create 方法
}

// Delete 删除用户
func (r *userRepository) Delete(id uint) error {
    return r.db.Delete(&model.User{}, id).Error  // 调用 gorm 的 Delete 方法
}

// Update 更新用户
func (r *userRepository) Update(user *model.User) error {
    return r.db.Save(user).Error  // 调用 gorm 的 Save 方法
}

// DeleteAll 删除所有用户
func (r *userRepository) DeleteAll() error {
    return r.db.Unscoped().Where("1=1").Delete(&model.User{}).Error  // 硬删除所有用户
}

// AddTestUser 添加测试用户
func (r *userRepository) AddTestUser() error {
    return r.db.Create(&model.User{
        Name:       "sleet",
        Account:    "943781228",
        Password:   "Zyz20050922!",
        Phone:      "13915181300",
        Avatar:     "https://example.com/avatar.jpg",
        Gender:     1,
        Birthday:   "2006-05-28",
        Location:   "江苏",
        UserStatus: 0,
    }).Error
}

// GetByID 根据 ID 获取用户
func (r *userRepository) GetByID(id uint) (*model.User, error) {
    var user model.User
    err := r.db.Where("id = ?", id).First(&user).Error
    if err != nil {
        return nil, err
    }
    return &user, nil
}

// GetByAccount 根据账号获取用户
func (r *userRepository) GetByAccount(account string) (*model.User, error) {
    var user model.User
    err := r.db.Where("account = ?", account).First(&user).Error
    if err != nil {
        return nil, err
    }
    return &user, nil
}

// GetByPhone 根据手机号获取用户
func (r *userRepository) GetByPhone(phone string) (*model.User, error) {
    var user model.User
    err := r.db.Where("phone = ?", phone).First(&user).Error
    if err != nil {
        return nil, err
    }
    return &user, nil
}

// UpdateField 更新用户指定字段
func (r *userRepository) UpdateField(userID uint, field string, value interface{}) (*model.User, error) {
    var user model.User
    err := r.db.Where("id = ?", userID).First(&user).Error
    if err != nil {
        return nil, err
    }
    err = r.db.Model(&user).Update(field, value).Error
    if err != nil {
        return nil, err
    }
    return &user, nil
}

// UpdateAvatar 更新用户头像
func (r *userRepository) UpdateAvatar(userID uint, avatar string) (*model.User, error) {
    return r.UpdateField(userID, "avatar", avatar)
}

// UpdateName 更新用户名
func (r *userRepository) UpdateName(userID uint, name string) (*model.User, error) {
    return r.UpdateField(userID, "name", name)
}
```

**讲解**：
- `UserRepository` 接口：定义了所有数据访问方法
- `userRepository` 结构体：实现了 `UserRepository` 接口
- `NewUserRepository` 构造函数：创建并返回 `userRepository` 实例
- 每个方法都实现了具体的数据访问逻辑
- `UpdateField` 方法：提取了重复的更新逻辑

### 第三步：创建业务逻辑层 (Service)

```go
package service

import (
    "errors"
    "sleet0922/graduation_project/internal/model"
    "sleet0922/graduation_project/internal/repo"
    "sleet0922/graduation_project/pkg/security"
    "strconv"
)

// UserService 业务逻辑接口
type UserService interface {
    Register(user *model.User) error                          // 注册用户
    DeleteAll() error                                        // 删除所有用户
    AddTestUser() error                                      // 添加测试用户
    Login(account, password string) (*model.User, error)      // 用户登录
    UpdateAvatar(userID uint, avatar string) (*model.User, error)  // 更新头像
    UpdateName(userID uint, name string) (*model.User, error)      // 更新用户名
}

// userService 业务逻辑实现
type userService struct {
    userRepo repo.UserRepository  // 依赖 UserRepository 接口
}

// NewUserService 创建 UserService 实例
func NewUserService(userRepo repo.UserRepository) UserService {
    return &userService{userRepo: userRepo}  // 注入 userRepo 依赖
}

// Register 注册用户
func (s *userService) Register(user *model.User) error {
    // 1. 密码加密
    hashedPassword, err := security.HashPassword(user.Password)
    if err != nil {
        return err
    }
    user.Password = hashedPassword
    
    // 2. 调用 Repository 层添加用户
    return s.userRepo.Add(user)
}

// DeleteAll 删除所有用户
func (s *userService) DeleteAll() error {
    return s.userRepo.DeleteAll()
}

// AddTestUser 添加测试用户
func (s *userService) AddTestUser() error {
    // 循环10次，创建10个测试用户
    for i := 0; i < 10; i++ {
        user := &model.User{
            Name:       "sleet" + strconv.Itoa(i),
            Account:    "test" + strconv.Itoa(i),
            Password:   "123456",
            Phone:      "13800000000" + strconv.Itoa(i),
            Avatar:     "https://example.com/avatar.jpg",
            Gender:     0,
            Birthday:   "2000-01-01",
            Location:   "北京",
            UserStatus: 0,
        }
        err := s.Register(user)  // 调用 Register 方法，会自动加密密码
        if err != nil {
            return err
        }
    }
    return nil
}

// Login 用户登录
func (s *userService) Login(account, password string) (*model.User, error) {
    // 1. 根据账号获取用户
    user, err := s.userRepo.GetByAccount(account)
    if err != nil {
        return nil, errors.New("账号或密码错误")
    }

    // 2. 验证密码
    err = security.CheckPassword(user.Password, password)
    if err != nil {
        return nil, errors.New("账号或密码错误")
    }

    return user, nil
}

// UpdateAvatar 更新头像
func (s *userService) UpdateAvatar(userID uint, avatar string) (*model.User, error) {
    return s.userRepo.UpdateAvatar(userID, avatar)
}

// UpdateName 更新用户名
func (s *userService) UpdateName(userID uint, name string) (*model.User, error) {
    return s.userRepo.UpdateName(userID, name)
}
```

**讲解**：
- `UserService` 接口：定义了所有业务逻辑方法
- `userService` 结构体：实现了 `UserService` 接口
- `NewUserService` 构造函数：创建并返回 `userService` 实例
- `Register` 方法：包含密码加密逻辑
- `Login` 方法：包含账号验证和密码检查逻辑

### 第四步：创建 HTTP 处理器 (Handler)

**文件**：`internal/handler/user_handler.go`

```go
package handler

import (
    "net/http"
    "sleet0922/graduation_project/internal/model"
    "sleet0922/graduation_project/internal/service"
    "sleet0922/graduation_project/pkg/jwt"
    "sleet0922/graduation_project/pkg/response"
    "time"

    "github.com/gin-gonic/gin"
)

// UserHandler HTTP 处理器
type UserHandler struct {
    userService service.UserService  // 依赖 UserService 接口
    jwtManager  *jwt.JWTManager      // 依赖 JWTManager 结构体
}

// NewUserHandler 创建 UserHandler 实例
func NewUserHandler(userService service.UserService, jwtManager *jwt.JWTManager) *UserHandler {
    return &UserHandler{
        userService: userService,
        jwtManager:  jwtManager,
    }
}

// Register 注册接口
func (h *UserHandler) Register(c *gin.Context) {
    // 1. 定义请求结构体
    type RegisterRequest struct {
        Name     string `json:"name" binding:"required"`     // 用户名，必填
        Account  string `json:"account" binding:"required"`  // 账号，必填
        Password string `json:"password" binding:"required"` // 密码，必填
        Phone    string `json:"phone" binding:"required"`    // 手机号，必填
    }

    // 2. 绑定请求参数
    var req RegisterRequest
    err := c.ShouldBindJSON(&req)
    if err != nil {
        response.Error(c, http.StatusBadRequest, "参数错误")
        return
    }

    // 3. 构建 User 模型
    user := &model.User{
        Name:     req.Name,
        Account:  req.Account,
        Password: req.Password,
        Phone:    req.Phone,
    }

    // 4. 调用 Service 层注册方法
    err = h.userService.Register(user)
    if err != nil {
        response.Error(c, http.StatusInternalServerError, "注册失败")
        return
    }

    // 5. 返回成功响应
    response.Success(c, user, "注册成功")
}

// DeleteAll 删除所有用户接口
func (h *UserHandler) DeleteAll(c *gin.Context) {
    err := h.userService.DeleteAll()
    if err != nil {
        response.Error(c, http.StatusInternalServerError, "删除所有用户失败")
        return
    }
    response.Success(c, nil, "删除所有用户成功")
}

// AddTestUser 添加测试用户接口
func (h *UserHandler) AddTestUser(c *gin.Context) {
    err := h.userService.AddTestUser()
    if err != nil {
        response.Error(c, http.StatusInternalServerError, "添加测试用户失败")
        return
    }
    response.Success(c, nil, "添加测试用户成功")
}

// Login 登录接口
func (h *UserHandler) Login(c *gin.Context) {
    // 1. 定义登录请求结构体
    type LoginRequest struct {
        Account  string `json:"account" binding:"required"`  // 账号，必填
        Password string `json:"password" binding:"required"` // 密码，必填
    }

    // 2. 绑定请求参数
    var req LoginRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.Error(c, http.StatusBadRequest, "参数错误")
        return
    }

    // 3. 调用 Service 层登录方法
    user, err := h.userService.Login(req.Account, req.Password)
    if err != nil {
        response.Error(c, http.StatusUnauthorized, err.Error())
        return
    }

    // 4. 生成 JWT token
    token, err := h.jwtManager.GenerateToken(user.ID, user.Account, time.Hour*24)
    if err != nil {
        response.Error(c, http.StatusInternalServerError, "生成token失败")
        return
    }

    // 5. 返回成功响应
    response.Success(c, gin.H{
        "token": token,
        "user": gin.H{
            "id":       user.ID,
            "account":  user.Account,
            "name":     user.Name,
            "avatar":   user.Avatar,
            "phone":    user.Phone,
            "gender":   user.Gender,
            "birthday": user.Birthday,
            "location": user.Location,
        },
    }, "登录成功")
}

// UpdateAvatar 更新头像接口
func (h *UserHandler) UpdateAvatar(c *gin.Context) {
    // 1. 定义更新头像请求结构体
    type UpdateAvatarRequest struct {
        ObjectKey string `json:"object_key" binding:"required"` // 对象键，必填
    }

    // 2. 绑定请求参数
    var req UpdateAvatarRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.Error(c, http.StatusBadRequest, "参数错误")
        return
    }

    // 3. 从上下文获取用户 ID
    userID, exists := c.Get("user_id")
    if !exists {
        response.Error(c, http.StatusUnauthorized, "未获取到用户信息")
        return
    }

    // 4. 调用 Service 层更新头像方法
    user, err := h.userService.UpdateAvatar(userID.(uint), req.ObjectKey)
    if err != nil {
        response.Error(c, http.StatusInternalServerError, "更新头像失败")
        return
    }

    // 5. 返回成功响应
    response.Success(c, gin.H{
        "id":         user.ID,
        "object_key": user.Avatar,
    }, "更新头像成功")
}

// UpdateName 更新用户名接口
func (h *UserHandler) UpdateName(c *gin.Context) {
    // 1. 定义更新用户名请求结构体
    type UpdateNameRequest struct {
        Name string `json:"name" binding:"required"` // 用户名，必填
    }

    // 2. 绑定请求参数
    var req UpdateNameRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.Error(c, http.StatusBadRequest, "参数错误")
        return
    }

    // 3. 从上下文获取用户 ID
    userID, exists := c.Get("user_id")
    if !exists {
        response.Error(c, http.StatusUnauthorized, "未获取到用户信息")
        return
    }

    // 4. 调用 Service 层更新用户名方法
    user, err := h.userService.UpdateName(userID.(uint), req.Name)
    if err != nil {
        response.Error(c, http.StatusInternalServerError, "更新用户名失败")

    }

    // 5. 返回成功响应
    response.Success(c, gin.H{
        "id":   user.ID,
        "name": user.Name,
```

- `UserHandler` 结构体：处理 HTTP 请求和响应

- 每个方法对应一个 HTTP 接口
- `Register` 方法：处理用户注册请求

```go
package router

import (
    "sleet0922/graduation_project/internal/config"
    "sleet0922/graduation_project/internal/handler"
    "sleet0922/graduation_project/internal/middleware"
    "sleet0922/graduation_project/internal/repo"
    "sleet0922/graduation_project/internal/service"
    "sleet0922/graduation_project/pkg/jwt"

    "github.com/gin-gonic/gin"
    "gorm.io/gorm"
)

func InitRouter(db *gorm.DB, cfg *config.ViperConfig) *gin.Engine {
    r := gin.Default()  // 创建 gin 引擎

    // 添加中间件
    r.Use(gin.Logger())  // 日志中间件
    r.Use(gin.Recovery())  // 恢复中间件

    // 初始化 JWT
    jwtManager := jwt.NewJWTManager(cfg.JWT.SecretKey)  // 创建 JWT 管理器
    jwtMiddleware := middleware.NewJWTMiddleware(jwtManager)  // 创建 JWT 中间件

    // 依赖注入
    userRepo := repo.NewUserRepository(db)  // 创建 UserRepository 实例
    userService := service.NewUserService(userRepo)  // 创建 UserService 实例，注入 userRepo
    userHandler := handler.NewUserHandler(userService, jwtManager)  // 创建 UserHandler 实例，注入 userService 和 jwtManager
    ossHandler := handler.NewOssHandler(cfg)  // 创建 OssHandler 实例

    // API 路由
    r.POST("/api/user/register", userHandler.Register)  // 注册接口
    r.POST("/api/user/login", userHandler.Login)  // 登录接口
    r.GET("/api/oss/upload-url", ossHandler.GetUploadURL)  // 获取上传 URL
    r.GET("/api/oss/download-url", ossHandler.GetDownloadURL)  // 获取下载 URL
    r.POST("/api/user/avatar", jwtMiddleware.Auth(), userHandler.UpdateAvatar)  // 更新头像（需要认证）
    r.POST("/api/delete_all", userHandler.DeleteAll)  // 删除所有用户
    r.POST("/api/add_test_user", userHandler.AddTestUser)  // 添加测试用户
    r.POST("/api/user/name_update", jwtMiddleware.Auth(), userHandler.UpdateName)  // 更新用户名（需要认证）

    return r
}
```

**讲解**：
- `InitRouter` 函数：初始化路由
- 中间件配置：添加日志和恢复中间件
- JWT 初始化：创建 JWT 管理器和中间件
- 依赖注入：按顺序创建和注入依赖
- 路由注册：将 HTTP 路径映射到对应的 handler 方法

## 三、核心概念深入讲解

### 1. 接口的作用

**接口**是 Go 语言中实现多态的关键。通过接口，我们可以：

- **定义行为规范**：接口定义了一组方法，任何实现了这些方法的类型都可以被视为该接口的实现
- **实现多态**：不同的类型可以实现同一个接口，在使用接口类型时，不需要关心具体的实现类型
- **解耦组件**：通过接口，我们可以将不同层的代码解耦，提高代码的可维护性

**示例**：
```go
// 定义一个接口
type Animal interface {
    Speak() string
}

// 狗实现 Animal 接口
type Dog struct {}

func (d Dog) Speak() string {
    return "Woof!"
}

// 猫实现 Animal 接口
type Cat struct {}

func (c Cat) Speak() string {
    return "Meow!"
}

// 使用接口
func makeAnimalSpeak(a Animal) {
    fmt.Println(a.Speak())
}

func main() {
    dog := Dog{}
    cat := Cat{}
    
    makeAnimalSpeak(dog)  // 输出: Woof!
    makeAnimalSpeak(cat)  // 输出: Meow!
}
```

### 2. 结构体的作用

**结构体**是 Go 语言中封装数据的基本单位。通过结构体，我们可以：

- **封装数据**：将相关的数据字段组织在一起
- **实现接口**：通过为结构体定义方法来实现接口
- **组织代码**：将相关的字段和方法组织在一起，提高代码的可读性

**示例**：
```go
// 定义一个结构体
type Person struct {
    Name string
    Age  int
}

// 为结构体定义方法
func (p Person) Greet() string {
    return fmt.Sprintf("Hello, my name is %s and I'm %d years old", p.Name, p.Age)
}

func main() {
    person := Person{Name: "Alice", Age: 30}
    fmt.Println(person.Greet())  // 输出: Hello, my name is Alice and I'm 30 years old
}
```

### 3. 构造函数的作用

**构造函数**是创建并返回结构体实例的函数。通过构造函数，我们可以：

- **初始化结构体**：确保结构体在创建时处于正确的状态
- **注入依赖**：将依赖的对象通过构造函数参数传入
- **提供创建实例的标准方式**：统一创建实例的接口，便于维护

**示例**：
```go
// 定义一个结构体
type Logger struct {
    prefix string
}

// 构造函数
func NewLogger(prefix string) *Logger {
    return &Logger{prefix: prefix}
}

// 方法
func (l *Logger) Log(message string) {
    fmt.Printf("[%s] %s\n", l.prefix, message)
}

func main() {
    logger := NewLogger("INFO")
    logger.Log("Hello, world!")  // 输出: [INFO] Hello, world!
}
```

### 4. 依赖注入的作用

**依赖注入**是一种设计模式，通过将依赖的对象通过构造函数或方法注入，而不是在内部创建。通过依赖注入，我们可以：

- **降低耦合**：减少组件之间的直接依赖
- **便于测试**：可以轻松替换依赖为 mock 对象
- **提高可维护性**：依赖关系清晰，便于理解和修改

**示例**：
```go
// 定义一个接口
type Database interface {
    Save(data string) error
}

// 实现一个数据库
type MySQLDatabase struct {
    // MySQL 相关配置
}

func (m *MySQLDatabase) Save(data string) error {
    fmt.Println("Saving to MySQL:", data)
    return nil
}

// 实现一个服务
type UserService struct {
    db Database  // 依赖 Database 接口
}

// 构造函数，注入依赖
func NewUserService(db Database) *UserService {
    return &UserService{db: db}
}

func (s *UserService) CreateUser(name string) error {
    return s.db.Save("User: " + name)
}

func main() {
    // 创建数据库实例
    db := &MySQLDatabase{}
    
    // 创建服务实例，注入数据库依赖
    service := NewUserService(db)
    
    // 使用服务
    service.CreateUser("Alice")  // 输出: Saving to MySQL: User: Alice
}
```

- 提供更具体的错误信息

```go
    }
    
    // 检查手机号是否已存在
```
